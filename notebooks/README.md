# Cadence Learning Notebooks

This folder is a beginner-friendly course for understanding how this repo is designed and how you could build it yourself.

The goal is not just to explain the code. The goal is to help you learn how to think like the person designing the code.

## Who this is for

- you are still new to software design
- you can read some code, but architecture still feels hard
- you want to understand both the web app and the Chrome extension
- you want to learn principles, not only copy code

## Best way to learn

- read one lesson at a time
- keep the repo open beside the lesson
- jump to the referenced files and trace the data flow
- answer the checkpoint questions before moving on
- try the exercises, even if they feel small

## Important mindset

You do not need to understand the whole repo at once.

You only need to understand:

- what problem a part solves
- what inputs it takes
- what outputs it produces
- what state it owns
- what other parts it talks to

If you can answer those five questions for a component, you are already doing system design.
